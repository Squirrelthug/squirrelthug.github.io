Q1. Integration with sounddevice Audio Stream

Use sounddevice.InputStream at 16 kHz mono and keep its callback lightweight. Silero works naturally with 30 ms chunks at 16 kHz, which is 480 frames; 60 ms is 960 frames. The callback should push numpy chunks into a queue. A worker thread performs VAD and utterance buffering, then hands completed utterances to a transcription queue for faster-whisper.

import queue, threading, time
import numpy as np
import sounddevice as sd
from silero_vad import load_silero_vad, VADIterator

SAMPLE_RATE = 16000
CHUNK = 480  # 30 ms
audio_q: queue.Queue[np.ndarray] = queue.Queue(maxsize=100)
utterance_q: queue.Queue[np.ndarray] = queue.Queue()
shutdown = threading.Event()

model = load_silero_vad()
vad = VADIterator(model, threshold=0.5, sampling_rate=SAMPLE_RATE,
                  min_silence_duration_ms=700, speech_pad_ms=250)

def input_callback(indata, frames, time_info, status):
    if status:
        pass  # record status outside callback if needed
    try:
        audio_q.put_nowait(indata[:, 0].copy())
    except queue.Full:
        pass

def vad_worker():
    speech_chunks = []
    recording = False
    while not shutdown.is_set():
        try:
            chunk = audio_q.get(timeout=0.1).astype(np.float32)
        except queue.Empty:
            continue
        event = vad(chunk, return_seconds=False)
        if event and "start" in event:
            recording = True
            speech_chunks = [chunk]
        elif recording:
            speech_chunks.append(chunk)
        if event and "end" in event and recording:
            audio = np.concatenate(speech_chunks)
            if len(audio) >= int(0.25 * SAMPLE_RATE):
                utterance_q.put(audio)
            recording = False
            speech_chunks = []

def transcribe_worker(whisper_model):
    while not shutdown.is_set():
        try:
            audio = utterance_q.get(timeout=0.1)
        except queue.Empty:
            continue
        segments, info = whisper_model.transcribe(audio, language="en")
        handle_transcript(" ".join(s.text for s in segments))

threading.Thread(target=vad_worker, daemon=True).start()
with sd.InputStream(samplerate=SAMPLE_RATE, channels=1, dtype="float32",
                    blocksize=CHUNK, callback=input_callback):
    while not shutdown.is_set():
        time.sleep(0.1)

Q2. End-of-Utterance Detection

Silero's threshold is the speech probability cutoff. Start with 0.5 for a home office and expose advanced tuning from roughly 0.35 to 0.65. End-of-utterance is controlled by silence duration, commonly min_silence_duration_ms=600-900. Use speech_pad_ms=200-300 to avoid clipped words and min_speech_duration_ms or an application-level duration check to ignore short noise bursts.

For natural thinking pauses, do not end on the first silent chunk. A 700 ms silence window is a practical default; voice assistants that need more interruption tolerance can use 900-1200 ms.

Q3. Latency Overhead

Silero advertises sub-millisecond CPU inference for one audio chunk on common CPUs. In practice, Python overhead plus queueing is still small relative to the 30 ms chunk size. Do not run inference inside the PortAudio callback thread: it is probably fast enough, but callback rules require avoiding blocking work and allocations. Added start latency is one chunk, about 30 ms. Added end latency is the configured silence duration plus one chunk, so about 730 ms with a 700 ms silence rule.

Q4. CPU vs GPU

Silero VAD is a small PyTorch/TorchScript model. CPU inference is fast enough that CUDA or Metal adds no meaningful benefit for Lilith's VAD stage and may add transfer overhead. It can run on CPU even if a CUDA-enabled PyTorch build is installed; explicitly keep model and tensors on CPU.

Q5. License

The silero-vad PyPI package declares MIT license classifiers. The upstream Silero VAD repository also publishes the VAD code/model under permissive terms; include MIT attribution in Lilith's legal notices. Commercial closed-source distribution is allowed with attribution.
