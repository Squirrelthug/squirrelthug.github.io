Q1. Linux installation with CUDA

To run faster-whisper with NVIDIA GPU/CUDA acceleration on Linux, install the package and its corresponding library bindings:

pip install faster-whisper
  • CUDA Requirements: CTranslate2 4.x requires **CUDA 12**. The base PyPI wheel does not bundle cuBLAS and cuDNN DLLs. To avoid system-wide CUDA SDK compilation mismatches, install the NVIDIA Python modules: 
    pip install nvidia-cublas-cu12 nvidia-cudnn-cu12
    We must then append the paths of these packages to LD_LIBRARY_PATH at application runtime in Linux.
  • Graceful Fallback: If CUDA is missing or version-mismatched, CTranslate2 raises a runtime error (e.g., failed to load shared library). It does *not* fall back to CPU automatically. The Lilith v6 application must catch the exception and fall back to CPU explicitly: 
    try:
    model = WhisperModel("small", device="cuda", compute_type="float16")
except Exception:
    # Fallback to CPU-only
    model = WhisperModel("small", device="cpu", compute_type="int8")

Q2. Apple Silicon support (macOS M1)

  • Hardware Acceleration Status: **faster-whisper does not support Apple Silicon GPU (Metal) or Apple Neural Engine (CoreML) acceleration.** The underlying CTranslate2 engine lacks MSL/CoreML backends for Whisper.
  • CPU Execution: On macOS arm64 (M1/M2/M3), faster-whisper runs strictly on the **CPU**. However, it is optimized utilizing ARM NEON and Apple's Accelerate Framework, delivering surprisingly fast performance.
  • WER & Latency: The CPU Word Error Rate (WER) is identical to GPU execution. For a 5-second voice utterance, the `small` model completes transcription in **~0.8 to 1.3 seconds** on M1 CPU, which is fully acceptable.
  • macOS Support Version: Stable arm64 macOS support was introduced in faster-whisper 0.6.0.

Q4. Model loading strategy

  • Memory Usage: The model is loaded into memory **once** at application startup during WhisperModel() initialization. It stays cached in VRAM/RAM for subsequent runs.
  • Loading Time: On standard SSDs, loading the `small` model takes **~1.5 seconds**, and the `medium` model takes **~4.0 seconds**.
  • Download & Setup: To avoid a long first-use delay, we should trigger model downloads during Lilith's installer/app setup process, rather than during the first voice query.
  • Cache Location: By default, models are cached in ~/.cache/huggingface/hub. This can be customized using Python's pathlib.Path and passing it to the download_root parameter: 
    model_dir = Path("~/projects/lilith/models/whisper").expanduser()
model = WhisperModel("small", download_root=model_dir)

Q5. Audio format requirements

  • Format: 16kHz, mono, 16-bit PCM (standard float32 array normalized between -1.0 and 1.0).
  • Direct Arrays: NumPy arrays returned by sounddevice can be passed directly to faster-whisper without file serialization.
  • Transcription Pattern: 
    import numpy as np
from faster_whisper import WhisperModel

# Instantiate model
model = WhisperModel("small", device="cpu", compute_type="int8")

# Transcribe float32 numpy array
segments, info = model.transcribe(
    audio_numpy_array,
    beam_size=5,
    language="en",
    vad_filter=True # secondary VAD pass inside Whisper
)

# Join segment fragments
transcribed_text = "".join([segment.text for segment in segments]).strip()

Q6. Streaming / chunked audio

  • Streaming Limitation: CTranslate2 is designed for batch processing. Real-time streaming transcription (processing audio chunks on the fly) is technically possible but unstable, resource-heavy, and prone to hallucinations.
  • Recommended Pattern (VAD-Gated Batching):
    1. sounddevice streams mic input to a queue in a background thread.
    2. Silero VAD monitors the audio stream.
    3. Once Silero VAD detects an "end of speech" trigger (silence threshold reached), the recording thread stops.
    4. The queue is compiled into a single NumPy array and dispatched to faster-whisper for a single, fast batch transcription pass.

Q7. Windows CUDA DLL issues

  • The Issue: CTranslate2 requires Windows-native DLLs (like cublas64_12.dll and cudnn_ops_infer64_8.dll) which are not bundled with faster-whisper or the system's standard NVIDIA graphics driver.
  • Resolution: We must bundle these DLLs inside the PyInstaller installation folder, or install the nvidia-cublas-cu12 and nvidia-cudnn-cu12 pip packages and use Python's os.add_dll_directory() to point to their paths at app startup: 
    import os
import sys
if sys.platform == 'win32':
    # Example paths for pip-installed nvidia dependencies
    os.add_dll_directory(r"C:\Path\To\Python\Lib\site-packages\nvidia\cublas\lib")
    os.add_dll_directory(r"C:\Path\To\Python\Lib\site-packages\nvidia\cudnn\lib")
