RES-13 — Argon2 Key Derivation

Researched: June 28, 2026 · argon2-cffi 25.1.0 · SQLCipher 4.x API

Q1. Argon2id Recommended Parameters

Use Argon2id. OWASP's floor is lower than what a desktop app can afford; for Lilith, target 100-500 ms on first startup and measure per platform. Recommended default: time_cost=3, memory_cost=65536 KiB (64 MiB), parallelism=2, hash_len=32. This uses about 64 MiB working memory per derivation plus overhead. If older hardware is slow, calibrate down to 32 MiB while keeping Argon2id.

Derive the six database keys sequentially, not concurrently, to avoid multiplying memory usage during startup.

Q2. Salt Storage

Use a per-database random 16- or 32-byte salt, stored in a small encrypted-or-integrity-protected metadata file under Lilith's config directory or in the database metadata record outside the key derivation dependency loop. A salt is not secret; it must be stable and backed up with the database. Do not store the salt only in keychain, because database restores would then fail without keychain state. Do not use one salt for all six databases, because per-database salts give domain separation even with one master secret.

SQLCipher's internal salt/header is for its own KDF path. If Lilith passes a raw derived key, keep Lilith's Argon2 salt explicit and versioned.

Q3. PRAGMA key vs PRAGMA hexkey

If Lilith derives a raw 32-byte key with Argon2id, pass it as a raw SQLCipher key using hex syntax: PRAGMA key = "x'...64 hex chars...'". This bypasses SQLCipher's passphrase PBKDF2 derivation and uses the supplied bytes as key material. Do not also spend time in SQLCipher's passphrase KDF. Set SQLCipher compatibility/cipher settings before first access as required by the chosen SQLCipher version, then key the database and verify with a trivial query.

conn = sqlcipher3.connect(str(db_path))
conn.execute(f"PRAGMA key = \"x'{hex_key}'\";")
conn.execute("PRAGMA journal_mode=WAL;")
conn.execute("PRAGMA foreign_keys=ON;")
conn.execute("SELECT count(*) FROM sqlite_master;")

Q4. Key Re-Derivation When Master Secret Changes

Changing the master secret means all six databases need new derived keys and must be re-encrypted. SQLCipher supports PRAGMA rekey on an open keyed database. Runtime depends on database size and disk speed; a 100 MB database is usually seconds, but Lilith should treat rekey as a blocking maintenance operation with progress and backup.

There is no atomic transaction spanning six separate database files. Pattern: create verified backups, rekey one database at a time, write a rotation manifest, fsync/verify each database, and recover from the manifest on next startup if interrupted.

Q5. argon2-cffi API

from __future__ import annotations
from argon2.low_level import Type, hash_secret_raw

ARGON2_TIME_COST = 3
ARGON2_MEMORY_COST_KIB = 65536
ARGON2_PARALLELISM = 2
KEY_LEN = 32

def derive_db_key(master_secret: bytes, db_salt: bytes, db_name: str) -> str:
    if len(db_salt) < 16:
        raise ValueError("Argon2 salt must be at least 16 bytes")
    domain_salt = db_salt + b"lilith-v6:" + db_name.encode("utf-8")
    key = hash_secret_raw(
        secret=master_secret,
        salt=domain_salt,
        time_cost=ARGON2_TIME_COST,
        memory_cost=ARGON2_MEMORY_COST_KIB,
        parallelism=ARGON2_PARALLELISM,
        hash_len=KEY_LEN,
        type=Type.ID,
    )
    return key.hex()

def open_sqlcipher(db_path, master_secret: bytes, salt: bytes, db_name: str):
    import sqlcipher3
    hex_key = derive_db_key(master_secret, salt, db_name)
    conn = sqlcipher3.connect(str(db_path))
    conn.execute(f"PRAGMA key = \"x'{hex_key}'\";")
    conn.execute("PRAGMA journal_mode=WAL;")
    conn.execute("PRAGMA busy_timeout=5000;")
    conn.execute("SELECT count(*) FROM sqlite_master;")
    return conn

Final recommendation: Use Argon2id with 64 MiB memory, time cost 3, parallelism 2, and per-database salts stored with Lilith database metadata/backups. Pass the 32-byte derived key through SQLCipher hexkey syntax. No blocker found, but benchmark startup derivation on low-end hardware.
