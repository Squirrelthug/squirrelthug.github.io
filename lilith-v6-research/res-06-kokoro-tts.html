Q2. CPU inference latency

Kokoro is highly optimized for CPU execution due to its small size (82M parameters). Below are latency estimates for synthesizing a typical 30-word conversational response (~10 seconds of speech):

Platform / Spec Real-Time Factor (RTF) Time to First Audio Sample Total Synthesis Time
MacBook M1 2020 (CPU-only, ARM) ~0.12 (8.3x real-time) ~150ms ~1.2 seconds
Intel Core i7-10700 (CPU-only, Linux) ~0.18 (5.5x real-time) ~220ms ~1.8 seconds
Low-spec Dual-Core CPU (Tier 0 baseline) ~0.40 (2.5x real-time) ~450ms ~4.0 seconds

An RTF of <1.0 means synthesis is faster than playback. Kokoro achieves an RTF of 0.12–0.18 on basic CPUs, making it ideal for Tier 0 local execution.

Q3. Streaming output

Yes, Kokoro natively supports streaming. The model decomposes text into clauses or sentences and yields synthesized audio chunks incrementally. Playback can begin as soon as the first chunk completes synthesis (approx. 150ms), reducing perceived latency.

Below is the Python streaming pipeline pattern:

from kokoro import KPipeline
import sounddevice as sd

# Initialize pipeline for English (US)
pipeline = KPipeline(lang_code='a')

text = "Hello! I am ready to assist. How can I help you today?"

# Generate audio stream (yields chunks incrementally)
audio_generator = pipeline(text, voice='af_heart', speed=1.0)

for graphemes, phonemes, audio in audio_generator:
    # Play each chunk dynamically as it is yielded
    sd.play(audio, samplerate=24000)
    sd.wait() # wait for chunk to finish before playing next

Q4. Python package and installation

  • PyPI Packages:
    • pip install kokoro — The official package. Requires PyTorch (which adds >2 GB to the installation footprint).
    • pip install kokoro-onnx onnxruntime — Recommended. Employs ONNX Runtime instead of PyTorch, reducing dependencies to ~50 MB, making it ideal for packaging closed-source client apps.
  • Model Download size: The Kokoro-82M ONNX model file is only **~300 MB**.
  • eSpeak NG Dependency: Kokoro requires the system package eSpeak NG for text-to-phoneme conversion.
    • Linux: Install via sudo apt install espeak-ng.
    • Windows/macOS packaging: To avoid forcing users to install eSpeak NG system-wide, we must bundle libespeak-ng.dll / .dylib and the espeak-ng-data lookup folder inside the PyInstaller distribution, setting os.environ["ESPEAK_DATA_PATH"] on application startup.

Q5. Real-time amplitude extraction for orb sync

To synchronize the 3D QML orb animation with Kokoro's voice output, we stream audio chunks to sounddevice and calculate the Root Mean Square (RMS) amplitude in real-time:

import numpy as np
import sounddevice as sd

def play_with_amplitude_sync(audio_data, sample_rate, qml_callback):
    block_size = 1024 # ~42ms chunks at 24kHz
    pointer = 0
    
    def callback(outdata, frames, time, status):
        nonlocal pointer
        remainder = len(audio_data) - pointer
        if remainder <= 0:
            outdata[:remainder] = 0
            raise sd.CallbackStop
            
        chunk_size = min(frames, remainder)
        chunk = audio_data[pointer:pointer+chunk_size]
        
        # Output assignment
        outdata[:len(chunk), 0] = chunk
        if len(chunk) < frames:
            outdata[len(chunk):, 0] = 0.0 # Pad remaining frames with silence
            
        pointer += chunk_size
        
        # Calculate RMS Amplitude
        rms = np.sqrt(np.mean(chunk ** 2)) if len(chunk) > 0 else 0.0
        # Normalize and push to QML bridge at ~60fps
        qml_callback(float(rms))

    # Open non-blocking sound stream
    stream = sd.OutputStream(channels=1, samplerate=sample_rate, 
                             blocksize=block_size, callback=callback)
    with stream:
        while stream.active:
            sd.sleep(10)

Q6. Voice selection

  • Voices: Kokoro offers numerous high-quality pre-trained voices, including female (af_heart, af_bella, af_nicole) and male (am_adam, am_michael, am_santa) personas.
  • Runtime Swapping: Voice selection can be changed instantly at runtime without reloading the model weights. The system simply swaps a 1x256 voice-style tensor.
  • Languages: Out-of-the-box support for English, Japanese, Mandarin Chinese, Spanish, French, and Portuguese.

Q7. GPU acceleration (for Tier 1/2 users)

  • Hardware Acceleration: Kokoro supports CUDA (via PyTorch or ONNX Runtime GPU) and Metal/CoreML (via ONNX Runtime execution providers).
  • Performance Gain: GPU execution accelerates Kokoro to an RTF of ~0.01 (100x real-time). However, since CPU-only inference already runs at sub-200ms latencies, the user-perceived difference is negligible. For Lilith v6, running Kokoro on the CPU across all tiers is fully viable and recommended.
Verdict: CLEAR (Using kokoro-onnx on CPU) Kokoro TTS is the ideal choice for Lilith v6. It is highly optimized, license-compatible (Apache 2.0), and provides natural-sounding voice synthesis locally on CPU.

Action Plan:
  1. Use the **`kokoro-onnx`** package to bypass the heavy PyTorch package dependency.
  2. Perform all TTS synthesis on the **CPU** across all hardware tiers. This keeps GPU VRAM free for local LLMs on Tiers 1 and 2.
  3. Employ the **sounddevice OutputStream callback model** to extract RMS amplitude in real-time, feeding it directly to the PySide6 UI bridge.
  4. **Bundle eSpeak NG binaries and data** inside the PyInstaller application bundle to ensure the app works on user machines without requiring separate system installs.
