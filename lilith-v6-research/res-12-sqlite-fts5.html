RES-12 — SQLite FTS5 in SQLCipher

Researched: June 28, 2026 · SQLCipher 4.x API · SQLite FTS5 module

Q1. FTS5 Availability in SQLCipher Python Builds

FTS5 is a SQLite compile-time option. SQLCipher can include it, but Python package wheels are inconsistent; Lilith must verify the actual bundled build at startup and in CI. The check is:

SELECT * FROM pragma_compile_options
WHERE compile_options LIKE 'ENABLE_FTS5%';

If no row is returned, building SQLCipher/sqlcipher3 from source with SQLite compiled using -DSQLITE_ENABLE_FTS5 is required. Prebuilt wheel availability for sqlcipher3 and pysqlcipher3 is not strong enough to treat FTS5 as guaranteed across Linux, macOS, and Windows. This should be part of RES-04 packaging acceptance tests.

Q2. FTS5 Schema for Conversation Search

Use external-content FTS5 so the encrypted canonical message table owns storage and the FTS table owns the index. All tables live inside the same SQLCipher database file, so the index is encrypted at rest.

CREATE TABLE messages (
    rowid INTEGER PRIMARY KEY,
    message_id TEXT NOT NULL UNIQUE,
    session_id TEXT NOT NULL,
    role TEXT NOT NULL CHECK (role IN ('user', 'assistant', 'system')),
    content TEXT NOT NULL,
    endpoint TEXT NOT NULL,
    created_at TEXT NOT NULL
);

CREATE VIRTUAL TABLE messages_fts USING fts5(
    content,
    role UNINDEXED,
    endpoint UNINDEXED,
    created_at UNINDEXED,
    content='messages',
    content_rowid='rowid',
    tokenize='unicode61'
);

CREATE TRIGGER messages_ai AFTER INSERT ON messages BEGIN
  INSERT INTO messages_fts(rowid, content, role, endpoint, created_at)
  VALUES (new.rowid, new.content, new.role, new.endpoint, new.created_at);
END;

CREATE TRIGGER messages_ad AFTER DELETE ON messages BEGIN
  INSERT INTO messages_fts(messages_fts, rowid, content, role, endpoint, created_at)
  VALUES ('delete', old.rowid, old.content, old.role, old.endpoint, old.created_at);
END;

CREATE TRIGGER messages_au AFTER UPDATE ON messages BEGIN
  INSERT INTO messages_fts(messages_fts, rowid, content, role, endpoint, created_at)
  VALUES ('delete', old.rowid, old.content, old.role, old.endpoint, old.created_at);
  INSERT INTO messages_fts(rowid, content, role, endpoint, created_at)
  VALUES (new.rowid, new.content, new.role, new.endpoint, new.created_at);
END;
-- Search with BM25 ranking.
SELECT m.message_id, m.session_id, m.role, snippet(messages_fts, 0, '<mark>', '</mark>', '...', 12) AS excerpt,
       bm25(messages_fts) AS rank
FROM messages_fts
JOIN messages m ON m.rowid = messages_fts.rowid
WHERE messages_fts MATCH :query
ORDER BY rank
LIMIT 20;

-- Surrounding context for a hit.
SELECT *
FROM messages
WHERE session_id = :session_id
  AND created_at BETWEEN datetime(:hit_time, '-10 minutes') AND datetime(:hit_time, '+10 minutes')
ORDER BY created_at;

Q3. FTS5 Performance at Lilith Scale

100,000 messages averaging 80 words is small for SQLite FTS5. Expect keyword BM25 queries to complete in milliseconds to low tens of milliseconds on a desktop SSD when the database is warm. The index increases database size because token postings are stored in addition to message text; external-content mode avoids a second full content copy, but postings and segment metadata still add meaningful overhead. Plan for roughly tens of percent extra size, then measure with realistic transcripts.

Q4. FTS5 External Content Trade-Off

Default FTS5 stores indexed text in the virtual table, which simplifies snippet queries but duplicates content. content='messages' external-content mode avoids duplicate message text and keeps the canonical row in one table, but requires triggers or application-level synchronization. content='' contentless mode minimizes storage further but makes updates/deletes and snippet generation more awkward.

Recommendation: external content. There is no SQLCipher-specific limitation because FTS pages are just database pages encrypted with the rest of the file.

Q5. Vector Search — sqlite-vec / sqlite-vss with SQLCipher

sqlite-vec is the successor-style lightweight vector extension path and is MIT/Apache-style permissive depending on component; verify exact bundled artifact before shipping. Runtime extension loading can work with SQLCipher if the Python binding exposes enable_load_extension(True) and load_extension(), and if the packaged SQLCipher build allows extension loading. Some hardened builds disable it.

Viable future path: keep FTS5 in the encrypted DB now, and later add an encrypted vector table with sqlite-vec if packaging tests pass. Do not choose a separate plaintext vector index.

Final recommendation: FTS5 inside SQLCipher is viable, but only if Lilith owns/verifies the SQLCipher build flags. Add a startup/CI assertion for ENABLE_FTS5. sqlite-vec is a plausible future path, with packaging and extension-loading tests required.
